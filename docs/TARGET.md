## 目标
1. 从算法视角，工程架构能够支撑 agent 的持续迭代，能够提供灰度发布、ab实验、模型参数管理、执行 trace
2. 从产品视角，支持基于基本的 agent 模式的自定义扩展，用户维度的 agent 隔离，用户维度的权限设置
3. 从工程视角，支持 agent 的服务发现，多agent实例的负载均衡、资源扩缩容，同步高优、异步填谷两条链路
4. 从稳定性视角，包含基本的限流、重试、监控、降级等工程特性
5. 从风控视角，支持回放、审计、拦截、过滤等风控能力
6. 从安全视角，支持敏感操作用户审批、恶意提示词注入识别等安全能力、代码执行沙箱环境
7. 从生态视角，支持 agent 的公开发布，tool 市场(用户编写、大模型生成)
